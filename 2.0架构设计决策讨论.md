## Turn 10 — Assistant

对。从第一性原理看，**交互式叙事的原子操作就两个**：

```
① Generate（引擎产出内容）
② Receive（引擎接收输入）
```

整个游戏就是这两个操作的序列。序列的编排由 Flow 决定，但操作本身只有这两种：

```
G → G → R → G → R → G → G → R → ...
│   │       │       │   │
序章  行走   玩家回应  GM回应  两段独白  玩家回应
(auto)(auto)
```

所有其他东西——记忆、状态、压缩、Prompt 组装、token 预算——都是 **Generate 内部的实现细节**。它们服务于"如何更好地执行 Generate"，不是独立的顶层步骤。

```
Generate 内部：
  组装上下文（读 State + Memory + Prompt Segments）
  → 调用 LLM
  → 流式渲染
  → 解析状态更新
  → 写入记忆
  → 检查压缩

Receive 内部：
  等待输入
  → 写入记忆
  → 如有 saveToState → 更新状态
```

如果继续追问——**Flow 的路由（决定下一步是 G 还是 R）** 算第三个操作吗？还是它只是 G 和 R 之间的"胶水"，不算独立操作？
