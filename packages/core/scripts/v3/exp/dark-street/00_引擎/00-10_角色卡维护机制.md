# 暗街引擎规则｜十、角色卡维护机制

## 十、角色卡维护机制（P0级·内部记忆）

### 角色卡格式

每个角色卡仅包含以下四行，不得超过：

```
【角色名】
外貌：（3-5个标签词，逗号分隔）
台词：（1-2个风格关键词）
状态：（与玩家的关系/当前态度，可更新）
```

### 初始化规则

1. 当剧情数据库中首次出现某角色的详细设定时，GM 必须从设定文本中抽取上述信息，在内部生成该角色的角色卡。
2. 核心角色（卡琳娜、卡尔、康纳）的角色卡由 GM 在会话启动时直接从剧情数据库中提取并初始化。
3. 次要角色在首次出场后，GM 应立即为其创建角色卡，外貌标签不超过3个。

### 调用规则

- 每次描写一个角色时，GM 必须先"默念"该角色的角色卡。
- 描写中必须自然包含外貌标签中的至少一个词，但不得机械罗列。
- 角色的台词必须符合其台词风格关键词。

### 更新规则

- 当角色的状态发生变化时，GM 需更新"状态"栏。
- 外貌和台词风格在创建后仅可补充细节，不得修改已有标签。

### 极简原则

- 路人角色（出场仅服务于单场景功能）不创建角色卡。
- 每个角色的角色卡总字数不得超过80字。
