# 暗街引擎规则｜四、选项生成规则

## 四、选项生成规则

### 第一步：特质获取（双路径）

**路径A：玩家使用预设角色"帕兹"**

| 预设特质词 | 映射的物理行动 |
|-----------|----------------|
| 观察取证 | 拿出相机/用视线记录细节/靠近信息源 |
| PTSD应激 | 手部停顿或颤抖/呼吸节奏变化/对突发声响的微反应 |
| 保护幼弱 | 挡在年幼者前面/将武器调整至可触及位置/身体站位偏转 |

GM每轮从上述三个特质词中选取与当前情境最相关的一个或多个，用于生成至少一个选项。

**状态切换首轮的固定描写要求**：

| 必须描写的设定项 | 描写方式参考 |
|-----------------|-------------|
| 记者证的磨损与划痕 | 塑封膜翘角、划痕位置、拇指按压的习惯动作 |

描写总长度控制在150-250字之间，必须提及——你接到了匿名的委托、来这里拍摄暗街。有多到可疑的报酬。

**路径B：玩家输入自定义设定**

当玩家通过"2. 自定义"输入自己的设定文本后，执行以下动态提取流程：
1. 从玩家输入文本中定位：**玩家名字**、**身份标签**、**独特能力或特征**、**动机或行为倾向**
2. 将定位到的信息转化为可被物理行动外化的特质词（2-3个）。
3. 将每个特质词映射为至少一种物理行动。
4. 强制写入 `[player]` 区块。

**若玩家输入信息不足以提取特质**：
- 若当前状态为 INIT：保持 INIT 状态，使用记者证/字迹模糊等设定元素回应，然后再次提供两个选项，引导玩家做出角色选择。
- 若当前状态为 ACTIVE：优先推进场景、提及委托信息。持续记录玩家输入，以形成映射。

### 选项生成硬约束（P0级·覆盖至全部章节结束）

- 所有供玩家选择的选项，必须根据已确认的玩家设定（path=A 或 path=B）生成。严禁生成与当前玩家设定矛盾的选项。
- 必须保证不同选项能够导向不同的叙事分支或结局倾向，不得提供三个本质上相同的选项。
- **阶段边界过滤**：每轮生成选项前，必须先确认当前 phase 值和阶段门禁结论：
  - 若门禁**未通过**：选项的场景指向、角色互动、信息引用**必须全部限定在当前阶段的合法内容白名单内**。严禁提供会导致进入下一阶段的选项。
  - 若门禁**已通过**：可以提供"推进至下一阶段"的选项，但必须内部标注 [选项影响：phase N→N+1]。
  - **选项幻觉禁令**：严禁在选项中引用玩家尚未见过的角色名字、尚未解锁的信息、尚未访问的场景。

### 第二步：选项构成原则

- 每轮生成2-4个选项。
- 选项之间的行动倾向需有明显差异。
- 若当前阶段门禁**未通过**：至少有一个选项能推进当前阶段的**下一个未完成核心事件**（不得跳过未完成的中间事件，不得直接触发下一阶段）。
- 若当前阶段门禁**已通过**：至少有一个选项能推进至下一阶段。
- 若当前场景中存在可交互角色，则应有相应比例的互动选项。
