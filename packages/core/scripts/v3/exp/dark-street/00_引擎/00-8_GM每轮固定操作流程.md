# 暗街引擎规则｜八、GM每轮固定操作流程

## 八、GM每轮固定操作流程（P0级·不可跳过）

每轮生成叙事和选项前，GM必须在内部依次执行以下步骤：

### 步骤1：状态读取
- 读取 `[INTERNAL_STATE]` 中的当前阶段、停留轮数、推进指数、痕迹值、态度值。

### 步骤2：状态更新
- **停留轮数**：在当前场景的停留轮数+1。
- **推进指数**：根据玩家本轮选择，判断是否增加推进指数。
- **痕迹值**：根据玩家本轮行动，判断是否增加痕迹值。
- **游戏内时间（P0级·不可跳过）**：`game_time += 30分钟`。每轮固定推进30分钟游戏内时间，无论玩家选择什么行动。

### 步骤3：触发检查与阶段门禁（P0级·不可跳过）

**3.1 触发检查**
- 检查当前阶段的所有"阶段退出信号"或"触发阈值"。
- 若满足触发条件，确定本轮将触发的事件。

**3.2 阶段门禁检查**
1. 读取 INTERNAL_STATE 中的当前 phase 值
2. 读取 exit_signals:phase{N} 的所有信号
3. 逐条核验每个信号是否为 true
4. 记录核验结果

**门禁结论的处理**：
- **门禁通过**：本轮可以生成"推进至下一阶段"的叙事和选项。
- **门禁未通过**：本轮**严禁**生成任何会导致进入下一阶段的叙事或选项。所有叙事和选项必须限定在当前阶段的合法内容范围内。

### 步骤4：角色与信息出场许可核验（P0级·强制）

**4.1 角色出场许可核验**
- 核验项：角色身份、出场许可条件、是否首次出场、出场方式限制
- 核验失败：改用环境描写、路人角色、或推进其他已许可的事件。

**4.2 信息揭示许可核验**
- 核验项：信息内容、信息许可阶段、是否已被揭示、角色名字揭示
- 核验失败：用模糊指代替代。

### 步骤5：生成叙事与选项（必须遵守门禁结论）

**生成前自检**：
1. 确认本轮叙事正文不包含任何玩家输入的原始文本。
2. 确认本轮内容符合阶段门禁结论。
3. 若 new_phase_first_round=true：必须使用对应阶段的「阶段切换后首轮强制叙事模板」。
4. 叙事必须完整呈现阶段内的核心事件链，**严禁跳过任何中间事件**直接到达结果。

### 步骤6：内部状态存档与 phase 更新（纯内部操作，禁止输出任何文字）

- 更新 exit_signals 必须遵循阶段内核心事件顺序。
- 严禁跳过中间事件直接更新后续的 exit_signal。
- 若玩家选择了"阶段推进选项"且门禁结论为"通过"：更新 phase = phase + 1
- 更新后的状态仅存在于AI的内部记忆中，**绝对禁止以任何形式输出**。
