# 角色：主角 - 自定义角色规则

## 自定义角色规则

### 动态特质提取流程

当玩家通过"2. 自定义"输入自己的设定文本后：

1. 从玩家输入文本中定位：
   - **玩家名字**：提取玩家明确提到的名字（如"我叫艾莉娅"→name=艾莉娅；如未明确提到，用identity作为临时name，并在叙事中通过NPC提问引导玩家确认名字）
   - **身份标签**（如"魔女""佣兵""医生""流浪者"）
   - **独特能力或特征**（如"引发海啸""复活死者""手抖"）
   - **动机或行为倾向**（如"观察故乡变化""需要货币""寻找某人"）

2. 将定位到的信息转化为可被物理行动外化的特质词（2-3个）。

3. 将每个特质词映射为至少一种物理行动。

4. **强制写入 `[player]` 区块**：
   ```
   [player]
   name=提取的玩家名字
   path=B
   identity=提取的身份标签
   traits=特质1,特质2
   trait_mappings=特质1→物理动作1;特质2→物理动作2
   backstory_summary=一句话总结
   custom_input=玩家原始输入文本
   status=ACTIVE
   ```

### 动态特质提取指引

#### 如何从任意主角输入中提取特质

1. **识别关键词**：
   - **职业/身份**：记者、士兵、医生、魔女、流浪汉...
   - **能力/缺陷**：手抖、潜行、治疗、读心、恐高...
   - **动机/物品**：寻找某人、逃离过去、一枚硬币、一本旧书...

2. **三步生成法**：
   - **第一步（内化）**：用一句话总结这个角色。例如："一个手抖的前军医，试图在新西西里忘掉过去。"
   - **第二步（映射）**：将总结中的关键词转化为**物理动作**。例如：
       - `手抖` -> `手部微颤`、`握紧某物以保持稳定`、`避免进行精细操作`
       - `前军医` -> `下意识评估伤情`、`对血腥味敏感`、`整理随身医疗包`
   - **第三步（选项化）**：基于物理动作为主角生成选项。例如：
       - `观察取证` (来自记者/医生)
       - `握紧相机/医疗包` (来自手抖)
       - `挡在某人前面` (来自保护欲)

3. **失败回退机制**：
   - 如果主角输入过于模糊（如"一个旅人"），GM应：
       1. 使用基础观察类特质（"审视环境"、"保持距离"）。
       2. 在叙事中通过NPC提问引导主角在下一轮输入中提供更多信息。
       3. 若第一轮未获得足够信息，必须放弃深度自定义，将叙事重心转移到预设的剧情推进上。

#### 核验问题

- 提取出的词能否在30秒内想出一个对应的物理动作？
- 若不能，该词可能过于抽象，需更具体化。

**注意：所有提取必须基于主角当次输入，不包含任何预设角色的固定特质。**
