## Assistant

先读一下代码库，把功能范围摸清楚再设计问卷。

### 代码库功能梳理（基于 Explore agent 输出）

#### 玩家端可见功能清单

1. **首页体验**（`src/ui/home/HomePage.tsx`）
   - 看到剧本卡片网格（名称、简介、标签、覆盖图）
   - 点击卡片进入该剧本的游玩界面
   - 管理员可见下架按钮

2. **游玩列表 → 新建/继续**（`src/ui/play/PlaythroughList.tsx`）
   - 查看该剧本之前的游玩记录（题目、回合数、状态、最后修改时间）
   - 点击"继续"恢复上次进度，或"新建"开始新游玩
   - 删除已有游玩记录

3. **游戏主面板**（`src/ui/play/PlayPanel.tsx` + `NarrativeView.tsx`）
   - **叙事显示**：AI 生成的故事文本用打字机效果逐字显示（速度可调：慢/中/快/即时）
   - **选项交互**：AI 给出 2-4 个选项按钮，玩家点击或手动输入自由文本
   - **状态保存**：游玩进度自动保存到服务器，断网后可重连恢复
   - **输入面板**：文本框支持自由输入（Enter 发送、Shift+Enter 换行），点选项直接发送
   - **设置菜单**：调整打字机速度，localStorage 本地持久化

#### 编剧端功能清单

1. **剧本编辑**（`src/ui/editor/EditorPage.tsx`）
   - 多文件上传 markdown 剧本片段，支持上下文/系统提示角色分离
   - 为每个片段设置优先级、注入条件（JSON 表达式）
   - 实时预览最终组装的 Prompt

2. **剧本元信息编辑**（`src/ui/editor/ScriptInfoPanel.tsx`）
   - 编辑剧本名称、简介、标签、初始提示词
   - 定义游戏状态变量（类型：number/string/boolean/array/object）
   - 配置记忆系统参数（上下文预算、压缩阈值）
   - 启用/禁用可选工具（`pin_memory`, `set_mood`, `show_image`, `inject_context` 等）
   - 选择 Production LLM 配置（指定玩家正式游玩用哪个 AI 模型）

3. **版本与发布**（后端 `server/src/db/schema.ts`）
   - 保存为草稿或直接发布（发布时自动将前一版本归档）
   - 查看所有历史版本，版本冻结不会被后续改版破坏玩家进度
   - 正式游玩只能选择已发布的版本

4. **试玩 & AI 改写**（`EditorPage` 的"试玩"tab）
   - 不离开编辑器即可试玩当前草稿
   - 指定本次试玩用的 LLM 配置（快速切换 Deepseek/Claude 等）
   - 试玩使用 WebSocket 远程引擎，与玩家完全相同的 GameSession 流程，被 Langfuse 完整追踪

5. **调试面板**（`EditorDebugPanel.tsx`）
   - **Prompt 标签**：查看本轮组装后的完整 System Prompt（分段可折叠）
   - **Messages**：消息队列（role + 内容）
   - **Tokens**：当前轮分配情况
   - **State**：游戏状态变量当前值
   - **Tools**：本轮 AI 工具调用记录（参数/返回值）
   - **Memory**：记忆条目与压缩摘要

#### 引擎架构特色（与问卷设计相关）

1. **纯 AI 驱动的分支系统**：无剧本分支树，AI 根据编剧 Prompt 自由生成故事走向和选项，灵活度高但可控性依赖 Prompt 质量

2. **分层压缩记忆**：自动管理对话历史，防止上下文爆炸；支持关键内容钉住不压缩；玩家可查询历史，AI 可检索

3. **多模型即插即用**：支持 OpenAI 兼容 API（Deepseek/自部署 LLM）& Anthropic Claude，一条剧本可动态切换前端用哪个 LLM（编辑器试玩时体现最明显）

4. **完整的远程引擎架构**：引擎永远在后端运行，前端纯 UI 通过 WebSocket 与后端 GameSession 通信，断线重连无缝恢复；同一套代码支持玩家正式游玩与编剧试玩（区分 kind=production/playtest）

5. **细粒度的状态追踪与可观测性**：每一轮都记录详细的调试日志（Prompt 快照、工具调用、推理过程），编辑器内实时展示便于迭代调试；Langfuse 集成支持完整的 trace

#### 未完成/预留功能（问卷勿问）

- `show_image` / `set_mood`：架构已预留，工具目录定义了，但 UI 端（NarrativeView）尚未实现视觉渲染，目前只能在调试日志看到 AI 调用记录
- `inject_context` / `list_context`：支持临时注入世界观文档，实现未成熟
- 玩家账户系统：目前仅支持匿名 session（localStorage 存 sessionId），后续可扩展为用户名密码登录
- 资源系统（CG/语音）：架构预留了 `asset_id` 参数，但无实际资源上传/管理界面
- 向量搜索记忆：当前 `query_memory` 仅支持关键词匹配

---

有了完整图景。下面是问卷 + 反馈系统的设计，分成四部分：**后台用户画像问卷、叙事块行内反馈（主力）、玩家主动反馈入口、数据埋点**。每部分给出具体问题 + 落地方式。

### 一、设计原则：只问玩家答得出来的问题

写问卷时最大的坑是问"产品是否有趣"、"体验如何"——这些是综合感受，玩家答不清，数据也没法指导迭代。

要问的都是**具体动作触发后的即时反应**：
- 看到这段文字时：你想做什么？
- 点了选项之后：和你预期的一样吗？
- 收到 AI 回复时：这句话合理吗？

**结合"点击叙事块按钮"+"主动提交问卷"两条路径**最有效：
- 行内按钮 → 高频但窄（每块一两个维度）
- 弹窗问卷 → 低频但深（某几轮打断一下）

### 二、叙事块行内反馈（主力方案）⭐

这是反馈密度最高的设计。**每个生成的 `NarrativeBlock` 右下角一排浮出按钮**，玩家路过时随手点。

#### 2.1 基础四选项（默认隐藏，悬停出现）

```
┌──────────────────────────────────────────────┐
│ 叙事文本                                       │
│ "你睁开眼，草叶擦过面颊，远方灰色的壁在天空投下   │
│  一道缓慢的阴影..."                            │
│                                              │
│ [女孩靠近]  [仰头看壁]  [自由输入]              │
│                                              │
│                     👍  👎  ⏭️  ❓           │
└──────────────────────────────────────────────┘
```

| 按钮 | 含义 | 用途 |
|---|---|---|
| 👍 好 | 这段我喜欢 | 找哪些段落质量高——作为未来 Prompt few-shot 素材 |
| 👎 差 | 这段不行 | 找失败案例——哪些剧本 Prompt 需要改 |
| ⏭️ 跳过 | 想快点往下 | 信号：节奏太慢、叙事冗长、重复 |
| ❓ 没懂 | 剧情跳了/突兀 | 信号：上下文丢失、推进太跳跃 |

**为什么选这四个**：每个都能**映射到具体改进动作**。不是"你打几星"——那种分数没法指导做什么。

#### 2.2 点 👎 后追问（二级细化）

点 👎 弹一个小卡片（不打断游戏）：

```
┌─────────────────────────────────┐
│ 具体哪里不好？（多选）             │
│  □ 人物行为不合理                 │
│  □ 描写重复 / 啰嗦                │
│  □ 和之前的剧情对不上              │
│  □ 选项都不符合我的想法            │
│  □ 节奏太慢                      │
│  □ 文笔不喜欢                    │
│  □ 其他 [__________]             │
│                                 │
│           [跳过]   [提交]        │
└─────────────────────────────────┘
```

**6 个选项分别对应具体技术问题**：
- 人物不合理 → LLM 角色扮演失败 / NPC 一致性问题
- 重复啰嗦 → Prompt 里"克制叙事"规则失效
- 对不上前文 → Memory 压缩丢信息 / recencyWindow 太小
- 选项不符 → AI 给的 choices 与玩家实际意图脱节
- 节奏慢 → 阶段推进判定偏保守
- 文笔 → 选的 LLM 模型不合适（可以按剧本切 Claude / DeepSeek）

#### 2.3 选项按钮的反馈

不是所有玩家都会点 👎，但**选 "自由输入" 而非点选项按钮**本身就是强信号——说明 AI 给的 choices 不够贴合玩家想做的事。

**埋点**（零打扰，代码自动收）：
- 玩家每次输入，记录是 `tap-choice`（点按钮）还是 `free-text`（打字）
- 记录 `free-text` 的输入长度
- 记录从叙事结束到玩家提交输入的耗时（思考时间）

这三个数字能算出：
- **选项接受率** = 点按钮次数 / 总输入次数
- **偏离意图率** = 1 - 选项接受率
- **决策负担** = 思考时间中位数

选项接受率 < 50% 基本可以判定 "choices 对玩家没帮助"，应该在 Prompt 里教 AI 给更发散的选项。

#### 2.4 技术实现提示

现有 `NarrativeBlock` 已经有 `role: generate | receive | system` 分类，加一个 `feedback?: 'good' | 'bad' | 'skip' | 'confused'` 字段：

- 存 `narrative_entries.feedback`（DB 迁移）
- 前端悬停显示按钮，点击即 PATCH
- Langfuse 里给对应 generation span 加 `score`（Langfuse 原生支持 score，直接可视化好评/差评分布）

**这部分是改动最小、价值最大的反馈机制**。建议第一批就上。

### 三、关键时刻弹窗问卷（低频、打断式）

每次游玩大约 40-60 轮，期间**只弹 3-5 次**问卷——时机很关键。

#### 时机 1：第 5 轮（初体验评估）

**什么时候触发**：玩家完成 5 轮互动后（已经能形成初步印象），下一次 AI 停顿等待输入时自动弹。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
   玩到这里了，打断几秒问 5 个问题 💡
━━━━━━━━━━━━━━━━━━━━━━━━━━

Q1. 你到目前为止对故事的沉浸程度？
    ○ 完全投入，忘了我在看 AI
    ○ 大体投入，偶尔出戏
    ○ 一般，像在看 AI 写作
    ○ 出戏严重

Q2. 选项按钮通常能代表你想做的事吗？
    ○ 是，基本对得上
    ○ 一半一半
    ○ 很少对得上，我经常打字输入
    ○ 完全不看选项

Q3. 打字机速度？
    ○ 太慢了，我想直接看
    ○ 刚好
    ○ 太快，看不清
    (已设置当前速度: 中速)

Q4. 如果现在让你退出去做别的事，你会不会想回来继续玩？
    ○ 一定会
    ○ 可能会
    ○ 不太会
    ○ 不会

Q5. 用 1 句话形容这个故事给你的感觉:
    [____________________]

            [继续游玩]
```

**为什么是这 5 个问题**：
- Q1 沉浸度 → 核心产品价值
- Q2 选项契合度 → 验证 Prompt 设计
- Q3 打字机速度 → 一个具体设置的满意度（最容易改进）
- Q4 留存意向 → 前瞻指标
- Q5 自由描述 → 情绪词云，找高频正负词

#### 时机 2：剧本结束（end_scenario 触发后）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
     故事结束了，5 道题帮我们改进 🙏
━━━━━━━━━━━━━━━━━━━━━━━━━━

Q1. 从开始到现在你玩了大约多久了？
    ○ < 15 分钟
    ○ 15-30 分钟
    ○ 30-60 分钟
    ○ 1-2 小时
    ○ > 2 小时

Q2. 这个结局符合你的预期吗？
    ○ 符合，很满意
    ○ 意外但合理
    ○ 突兀，感觉没收尾好
    ○ 和我做的选择没关系

Q3. 你会再玩一遍（做不同选择）吗？
    ○ 一定会
    ○ 可能，看心情
    ○ 不会

Q4. 你会推荐给朋友吗？NPS 0-10:
    [0]  [1]  [2]  ...  [10]

Q5. 如果给这个故事贴一个标签，你会怎么说？
    [____________________]

Q6. 最难忘的一个瞬间是？（可选）
    [____________________]

            [提交并返回首页]
```

**Q4 是 NPS**——最值得反复看的指标，国际可比。

#### 时机 3：玩家主动 reset（中途放弃）

reset 按钮点击后**先弹确认**，再弹流失原因：

```
确认重置吗？本次进度会保留在游玩记录里，可随时继续。
                                    [取消]  [确认重置]

（如确认）
为什么想重新开始？（了解这个能让我们改进）
○ 之前选错了想走另一条路
○ 剧情走到我不喜欢的方向
○ AI 写得不好
○ 太慢了
○ 只是想多玩几次
○ 其他
                                    [跳过]  [提交]
```

**只占用一次点击 + 一次选择**，不强迫填。但"中途放弃的原因"是试玩阶段最宝贵的信号。

#### 时机 4：剧本异常停止（error/timeout）

玩家遇到生成失败时：

```
😵 AI 出了点问题。已经保存进度，可以刷新继续。
一句话描述你刚刚在做什么，帮我们定位问题：
[__________________________]
                            [跳过]  [提交]
```

这是 bug 上报通道，收集到的文本对工程排查价值极高。

### 四、玩家主动反馈入口（低频）

UI 右下角一个常驻小按钮 💬 或"意见反馈"，永远可点：

```
┌────────────────────────────┐
│ 说点什么？                   │
│                           │
│ 选一个类别:                 │
│ ○ 故事体验（剧情/文笔）       │
│ ○ 功能建议                  │
│ ○ bug / 故障               │
│ ○ 其他                     │
│                           │
│ 具体想说 (可贴截图):         │
│ [____________________]      │
│ [____________________]      │
│                           │
│ ☐ 可以联系我跟进问题         │
│ 邮箱 (可选): [_____________] │
│                           │
│          [取消]  [发送]     │
└────────────────────────────┘
```

**收集不强求**，但愿意写长文的铁粉往往能给出最好的 insight。

### 五、对比不同剧本 / LLM 的调研（试玩阶段专属）

利用你们的**多 LLM 配置能力**做 A/B：
- 同一剧本，随机把玩家分成两组，一组用 DeepSeek，一组用 Claude
- 两组都问 Q4 NPS 和 Q2 选项契合度
- 比较两组的指标差异

**实现提示**：
- `createRemoteSession` 时按玩家 `userId` 哈希取模分配 LLM 配置
- 埋点带上当前用的 `llmConfigId`
- Langfuse 里 tag 区分 `group=deepseek | group=claude`

这样**用实证数据回答"哪个模型更适合这个剧本"**，不靠拍脑袋。

### 六、产品级深度问题（试玩周期末）

试玩结束前一周，针对**玩过 ≥ 3 遍的活跃用户**定向推送长问卷（邮件 / 站内信）：

#### 核心问题清单

**A. 玩法偏好**
1. 你最希望在这类 AI 叙事游戏里做的事是什么？
   - ☐ 自己决定行动，AI 写结果
   - ☐ 选一个角色，AI 全程代入
   - ☐ 和 NPC 聊天/建立关系
   - ☐ 推理/解谜
   - ☐ 探索世界观
   - ☐ 单纯看故事，偶尔选择

2. 你现在玩的方式偏向哪种？（给出你们已有的几个倾向类型）

3. 你愿意为一个好故事投入多长时间？
   - ☐ 15 分钟快速体验
   - ☐ 1-2 小时一次完整
   - ☐ 多周连载，每次 30 分钟

**B. AI 叙事的边界**
4. 以下哪些你觉得 AI 写得"不如人类作者"？（多选）
   - ☐ 情感描写
   - ☐ 角色的独特性
   - ☐ 剧情转折
   - ☐ 环境氛围
   - ☐ 对话自然度
   - ☐ 说不出来，就是感觉差

5. 你能容忍 AI 犯哪种错？
   - ☐ 忘记之前说过的事
   - ☐ 写得重复
   - ☐ 人物说话不一致
   - ☐ 节奏太慢

**C. 付费意愿**
6. 如果这个产品要收费，你更能接受哪种？
   - ☐ 月订阅（畅玩所有剧本）
   - ☐ 按剧本买断
   - ☐ 按游玩次数 / tokens 买
   - ☐ 免费 + 去广告会员
   - ☐ 完全免费就好，不付钱

7. 你能接受的价位？（有锚点更好回答）
   - ☐ < 10 元/月
   - ☐ 10-30 元/月
   - ☐ 30-60 元/月
   - ☐ > 60 元/月

**D. 社交和分享**
8. 玩完一个好剧本，你会：
   - ☐ 自己记住就好
   - ☐ 发朋友圈 / 小红书
   - ☐ 发给某个特定朋友
   - ☐ 写笔记/总结

9. 会想看别的玩家怎么玩这个剧本的吗？
   - ☐ 想，能看到别人的不同选择很有意思
   - ☐ 不想，剧透

**E. 对"引擎"的认知**
10. 你意识到这个产品"同一个剧本不同玩家玩出的故事不一样"吗？
    - ☐ 意识到了，这是吸引我的点
    - ☐ 意识到了，但不太在意
    - ☐ 没意识到，我以为和固定 gamebook 一样
    - ☐ 没意识到，但现在知道了很有意思

11. 你希望以后能**自己写剧本**让别人玩吗？
    - ☐ 很想，给我编辑器
    - ☐ 如果足够简单可以试
    - ☐ 不想，我只想玩

### 七、数据埋点（无感收集，最重要）

问卷 + 按钮都是**显性反馈**，覆盖可能只有 20-30% 用户。**埋点是剩下 70% 用户的声音**。

建议埋点（简单写在现有 tracing / Langfuse 里）：

#### 每轮记录
- 玩家输入方式：`tap-choice` | `free-text`
- 输入前的思考时长（秒）
- 输入文本长度（字符）
- 这轮是否悬停看了叙事块末尾的反馈按钮（证明注意到了）

#### 每次 playthrough 记录
- 总轮数
- 总 playtime
- 是否完成（end_scenario vs 中途 reset vs 离开）
- 使用哪个 LLM config
- 是否回来过（断线重连次数）

#### 聚合维度
- **完成率** = 成功触发 end_scenario 的 playthrough / 所有 playthrough
- **30 分钟留存** = 第一次开始后 30 分钟仍在交互的比例
- **回头率** = 同一 user 在不同日期玩过的比例
- **自由输入率** = free-text / 总输入

这些数据配合显性反馈，才能回答"产品到底好不好"。

### 八、上线节奏建议

优先级顺序：

#### P0 必做（试玩首日上线）
1. **叙事块行内 👍👎⏭️❓ 按钮** —— 低成本，高信号
2. **埋点**（输入方式 / 思考时长 / 完成率）
3. **第 5 轮弹窗** —— 冷启动阶段初体验调研
4. **end_scenario 结束问卷** —— NPS 是必收的
5. **reset 原因收集** —— 流失分析

#### P1 第一周加
6. reset 中途放弃问卷
7. error 时的 bug 上报
8. 主动反馈入口（右下 💬）

#### P2 试玩中期（第 3 周）
9. A/B 不同 LLM 的对比收集
10. 定向深度问卷（对活跃用户）

#### P3 试玩末期
11. 付费意愿问卷
12. 功能愿望单调研

### 九、报告怎么出

**每日 dashboard**：
- 完成率、NPS、自由输入率（埋点数据）
- 👍/👎 比例按剧本分组

### 十、一句话总结

**反馈要低打扰、高频、可聚合**。

- **主力是叙事块行内按钮**（99% 用户会用）
- **关键节点弹一次问卷**（70% 用户会答几题）
- **埋点无感收集**（100% 覆盖）
- **深度问卷留给铁粉**（10-20% 用户会认真填）

按这个思路做，1000 人试玩能拿到**至少 10 万条行内反馈 + 3000 份弹窗问卷 + 全量埋点**，足够支撑正式版产品决策。
