# IVN 互动视觉小说引擎 — 编剧使用指南

> 最后更新：2026-04-16

---

## 一、概览

IVN 引擎运行在**远程模式**下：后端统一管理 LLM 调用、游玩记录和剧本数据。

- **管理员**（admin）：使用编辑器编写/测试/发布剧本，管理 LLM 配置
- **玩家**：在首页选择已发布的剧本进行游玩

> 注意：v2.6（6.6）之前曾支持浏览器本地模式（IndexedDB + 直连 LLM），该模式已下线。

---

## 二、首页

首页展示所有已发布的剧本卡片。每张卡片显示：

- **剧本名称**（顶部大字）
- **简介**（自适应高度，尽量显示完整）
- **标签**（如"科幻""治愈"，最多显示 3 个）
- **版本号**（左下角，如 v3）
- **章节数**（左下角）
- 如果有封面图，会作为卡片背景

**点击卡片** → 进入游玩页面，开始互动。

### 管理员功能

- 右上角显示用户名和"退出"按钮
- 出现"编辑器"按钮，点击进入编辑器
- 鼠标悬浮在卡片上会出现红色**"下架"**按钮

---

## 三、管理员登录

首页上**没有**登录按钮，请使用快捷键：

> **Ctrl + Shift + L**（Mac 上是 Control + Shift + L）

会弹出登录弹窗，输入用户名和密码后点击"登录"。

管理员账号通过 seed 脚本从环境变量创建，详见 `server/scripts/seed-admin.ts`。
**请勿在代码或文档中硬编码密码。** 初始密码请在首次部署后立即修改。

登录后可以：
- 进入编辑器
- 发布/下架剧本
- 管理 LLM 配置（多套命名配置）

---

## 四、编辑器

### 4.1 进入方式

先用 Ctrl+Shift+L 登录管理员，然后点首页的"编辑器"按钮。

### 4.2 整体布局

```
┌─────────────────────────────────────────────────────┐
│  ← 返回   编剧编辑器 — 剧本名    文件数 | tokens | 发布  │
├──────────────┬──────────────────────────────────────┤
│  左侧面板     │         右侧 Tab 面板                  │
│  文件列表      │  Prompt预览 | 试玩 | 调试 | 剧本信息 | 设置 │
│  + 代码编辑器   │                                      │
│  + Initial    │                                      │
│    Prompt     │                                      │
└──────────────┴──────────────────────────────────────┘
```

### 4.3 左侧面板 — 文件管理

#### 新建/打开剧本

- 左上角下拉菜单可以切换已保存的剧本
- **保存**按钮（绿色）：保存当前剧本到后端，创建新版本
- 菜单中还有：新建、导入（.ivn.json）、导出、删除

#### 上传文件

- 点击"上传 .md"按钮，或直接**拖拽 .md 文件**到左侧区域
- 支持 .md 和 .txt 格式
- 上传后自动识别文件角色：
  - 文件名含 `gm_prompt` 或 `GM prompt` → 标记为 **system**（系统 prompt）
  - 文件名含 `pc_prompt` 或 `PC prompt` → 标记为 **context**（角色设定）
  - 其他 → 标记为 **draft**（草稿）
- 也能自动识别章节：文件名含"第一章""序章"等会自动设置注入条件

#### 编辑文件

- 点击文件列表中的文件名，右侧（或中间区域）会打开编辑器
- 编辑器上方有文件元数据栏：
  - **Role**（角色）：system / context / draft
  - **Priority**（优先级）：数字越大越靠后
  - **Condition**（注入条件）：为空表示始终注入，也可以写 `chapter === 2` 这样的表达式
  - **Description**（描述）：备注用

#### Initial Prompt

- 左侧底部有一个 **INITIAL PROMPT** 输入框
- 这是游戏开始时发送给 AI 的第一条消息
- 默认是"开始测试"，你可以改成更具体的引导语

### 4.4 右侧 Tab — Prompt 预览

展示最终发给 AI 的 prompt 是怎么组装的：

- **Token 使用量**：彩色进度条，显示 system/context/engine 各占多少
- **Assembly Order**：所有 prompt 片段的排列顺序
  - 可以**拖拽调整顺序**
  - "自动排序（优化缓存）"按钮会把稳定内容放前面，动态内容放后面，以优化 API 前缀缓存
  - "重置默认"恢复初始顺序
- 点击每个片段可以展开查看完整内容
- 灰色的 `engine` 标记表示引擎自动生成的部分（状态 YAML、记忆摘要、历史记录、引擎规则等），不需要你手动维护

### 4.5 右侧 Tab — 试玩

在编辑器里直接测试你的剧本（通过后端 WebSocket 连接 LLM）：

1. 在上方选择**试玩使用的 LLM 配置**（下拉菜单，从 LLM 配置管理中选）
2. 点击**"开始"**按钮启动游戏
3. 等待 AI 生成开场叙事
4. 在底部输入框写你的行动/对话/想法，按回车提交
5. AI 会根据你的输入继续叙事
6. 可以随时点**"重置"**开始新的试玩

### 4.6 右侧 Tab — 调试

开发调试用，有 6 个子面板：

1. **Prompt**：查看最终组装的 system prompt 全文
2. **Messages**：查看完整对话消息列表（含 role）
3. **Tokens**：Token 分配明细
4. **State**：当前所有状态变量和值
5. **Tools**：已注册的工具和最近的工具调用记录
6. **Memory**：记忆条目和压缩事件

### 4.7 右侧 Tab — 剧本信息

编辑剧本的元数据：

#### 基本信息
- **剧本名称**：显示在首页卡片和编辑器标题栏
- **简介**：显示在首页卡片上，建议写得详细一些
- **Initial Prompt**：同左侧面板底部的那个
- **生产使用 LLM**：选择玩家正式游玩时使用的 LLM 配置

#### 标签
- 在输入框里打字，按**回车**或点 **+** 添加标签
- 点标签右边的 **×** 删除
- 标签显示在首页卡片上，帮助玩家了解剧本类型

#### 状态变量
- 定义剧本用到的状态变量（如 chapter、stage、mood 等）
- 每行包含：变量名 | 类型 | 初始值 | 描述
- 类型支持：number、string、boolean、array、object
- 点"+ 添加变量"增加新变量，点 × 删除

#### 记忆配置
- **Token 预算**：总的上下文窗口预算
- **压缩阈值**：对话记忆超过这个 token 数时触发压缩
- **最近保留条数**：压缩时保留最近几条不压缩

#### 启用工具
- 勾选 AI 可以使用的工具（详见下方"工具系统详解"章节）

### 4.8 右侧 Tab — 设置（LLM 配置管理）

管理多套 LLM 连接配置（v2.7 起存储在后端数据库）：

- **新增配置**：点 "+" 按钮，填写名称、Provider、Base URL、API Key、模型名
- **编辑/删除**：每行右侧的操作按钮
- **每套配置**还可设置：思考模式开关、推理过滤器开关、max output tokens

> 编辑器试玩和玩家正式游玩可以使用不同的 LLM 配置。

### 4.9 发布剧本

- 顶栏右侧有**"发布"**按钮（绿色）
- 必须先保存过一次（有 script ID）才能发布
- 发布会把完整剧本快照存入后端的 script_versions 表
- 同一剧本同时只能有一个 published 版本，旧版自动归档

## 五、游玩页面

点击首页卡片进入：

- 顶部显示剧本名称和返回按钮
- 如果有之前的游玩记录，可以选择继续或开始新游戏
- AI 生成的叙事内容以流式方式逐字显示（打字机效果）
- 在底部输入框写你的行动/对话/想法，按回车提交
- 通过 WebSocket 连接后端，后端负责调 LLM API

## 六、工具系统详解

引擎为 AI 提供了一套工具（Tools），AI 在生成叙事时可以主动调用它们来管理游戏状态、记忆、UI 等。你在编辑器"剧本信息"tab 的"启用工具"中可以选择开放哪些工具给 AI 使用。

### 必选工具（始终启用，不可关闭）

#### `update_state` — 更新游戏状态

AI 用这个工具来修改状态变量。比如玩家到达第 2 章时，AI 会调用 `update_state({"chapter": 2})`。

- **用途**：推进剧情进度、记录玩家选择、更新角色属性等
- **参数**：一个 JSON 字符串，包含要更新的变量名和新值
- **编剧需要做的**：在"状态变量"中定义好变量（如 chapter、stage、trust_level），AI 会根据你的 prompt 来决定何时修改它们

#### `signal_input_needed` — 等待玩家输入

AI 生成一段叙事后，调用这个工具暂停并等待玩家输入。这是引擎核心循环的关键。

- **用途**：叙事到达一个需要玩家做出反应的节点时触发
- **可选功能**：AI 可以提供几个建议选项（choices），玩家可以点选或自由输入

#### `end_scenario` — 结束剧情

AI 认为故事已经走完（达成结局、所有剧情线收束）时调用。调用后 playthrough 标为 finished，玩家界面不再接受输入。

- **用途**：标记故事的真正终点
- **注意**：不可逆——一旦调用，当前 session 无法继续。场景切换、暂时离开等请用 `signal_input_needed`

### 可选工具（在"启用工具"中勾选）

#### `read_state` — 读取状态变量

AI 需要查看当前游戏状态时调用。推荐大多数剧本启用。

#### `query_changelog` — 查询状态变更日志

查看某个状态变量的历史变化。较复杂的剧本（有分支记忆、需要回顾历史决策）建议开启。

#### `pin_memory` — 钉住重要记忆

标记一段内容为"重要记忆"，在记忆压缩时不会被丢弃。

#### `query_memory` — 搜索记忆

用关键词搜索历史记忆。配合 `pin_memory` 使用效果最好。

#### `set_mood` — 设置场景氛围

设置当前场景的情绪/氛围标签。

### 工具在 prompt 中的引用

在编剧文档中直接写工具裸名（如 update_state、signal_input_needed），GM 会从 tool schema 识别这些名称。

## 七、典型工作流

### 编剧创建新剧本

1. 进入编辑器
2. 上传你写好的 .md 文件（GM prompt、世界观、角色设定等）
3. 在文件列表里检查每个文件的 Role 和 Priority 是否正确，不对就手动调
4. 切到"剧本信息"tab，填写名称、简介、标签、选择生产 LLM 配置
5. 检查/调整状态变量和记忆配置
6. 切到"试玩"tab，选择试玩用的 LLM 配置，点"开始"测试
7. 测试满意后，点"保存"，再点"发布"

### 更新已发布剧本

1. 在编辑器打开已有剧本
2. 修改文件内容或设置
3. 点"保存"（自动创建新版本）
4. 重新点"发布"

## 九、常见问题

**Q: 试玩时报错或无响应？**
A: 检查 LLM 配置是否正确（设置 tab），特别是 Base URL 和 API Key。

**Q: 发布按钮是灰色的不能点？**
A: 需要先点"保存"，让剧本有一个 ID 之后才能发布。

**Q: 看不到"编辑器"按钮？**
A: 需要先用 Ctrl+Shift+L 登录管理员账号。

**Q: 版本管理怎么运作？**
A: 每次保存自动创建新版本。同一剧本只有一个 published 版本，旧版自动归档。玩家的老 playthrough 继续使用创建时那个版本的 manifest。
