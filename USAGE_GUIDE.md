# IVN 互动视觉小说引擎 — 编剧使用指南

> 最后更新：2026-04-03

---

## 一、概览

IVN 引擎分为两种运行模式：

- **本地模式（Local）**：一切在浏览器内运行，你一个人就能写剧本、测试、发布。需要自己配置 LLM API Key。
- **远程模式（Remote）**：后端统一管理，多人共用。需要管理员账号才能使用编辑器和发布功能，普通玩家只能在首页选剧本来玩。

---

## 二、首页

首页展示所有已发布的剧本卡片。每张卡片显示：

- **剧本名称**（顶部大字）
- **简介**（自适应高度，尽量显示完整）
- **标签**（如"科幻""治愈"，最多显示 3 个）
- **版本号**（左下角，如 v0.0.4）
- **章节数**（左下角）
- 如果有封面图，会作为卡片背景

**点击卡片** → 进入游玩页面，开始互动。

### 管理员功能（远程模式）

- 右上角显示用户名和"退出"按钮
- 出现"编辑器"按钮，点击进入编辑器
- 鼠标悬浮在卡片上会出现红色**"下架"**按钮（右上角），点击后会弹出确认框，确认后该剧本从首页移除（后端直接删除，要重新发布时从编辑器再推送一次即可）

## 四、编辑器

### 4.1 进入方式

- **本地模式**：首页直接有"编辑器"按钮
- **远程模式**：先用 Ctrl+Shift+L 登录管理员，然后点"编辑器"

### 4.2 整体布局

```
┌─────────────────────────────────────────────────────┐
│  ← 返回   编剧编辑器 — 剧本名    文件数 | tokens | 发布  │
├──────────────┬──────────────────────────────────────┤
│  左侧面板     │         右侧 Tab 面板                  │
│  文件列表      │  Prompt预览 | 试玩 | 调试 | 剧本信息 | 设置 │
│  + 代码编辑器   │                                      │
│  + Initial    │                                      │
│    Prompt     │                                      │
└──────────────┴──────────────────────────────────────┘
```

### 4.3 左侧面板 — 文件管理

#### 新建/打开剧本

- 左上角下拉菜单可以切换已保存的剧本
- **保存**按钮（绿色）：保存当前剧本到本地 IndexedDB，版本号自动 +1（patch 位）
- 菜单中还有：新建、导入（.ivn.json）、导出、删除

#### 上传文件

- 点击"上传 .md"按钮，或直接**拖拽 .md 文件**到左侧区域
- 支持 .md 和 .txt 格式
- 上传后自动识别文件角色：
  - 文件名含 `gm_prompt` 或 `GM prompt` → 标记为 **system**（系统 prompt）
  - 文件名含 `pc_prompt` 或 `PC prompt` → 标记为 **context**（角色设定）
  - 其他 → 标记为 **draft**（草稿）
- 也能自动识别章节：文件名含"第一章""序章"等会自动设置注入条件

#### 编辑文件

- 点击文件列表中的文件名，右侧（或中间区域）会打开编辑器
- 编辑器上方有文件元数据栏：
  - **Role**（角色）：system / context / draft
  - **Priority**（优先级）：数字越大越靠后
  - **Condition**（注入条件）：为空表示始终注入，也可以写 `chapter === 2` 这样的表达式
  - **Description**（描述）：备注用

#### Initial Prompt

- 左侧底部有一个 **INITIAL PROMPT** 输入框
- 这是游戏开始时发送给 AI 的第一条消息
- 默认是"开始测试"，你可以改成更具体的引导语

### 4.4 右侧 Tab — Prompt 预览

展示最终发给 AI 的 prompt 是怎么组装的：

- **Token 使用量**：彩色进度条，显示 system/context/engine 各占多少
- **Assembly Order**：所有 prompt 片段的排列顺序
  - 可以**拖拽调整顺序**
  - "自动排序（优化缓存）"按钮会把稳定内容放前面，动态内容放后面，以优化 API 前缀缓存
  - "重置默认"恢复初始顺序
- 点击每个片段可以展开查看完整内容
- 灰色的 `engine` 标记表示引擎自动生成的部分（状态 YAML、记忆摘要、历史记录、引擎规则等），不需要你手动维护

### 4.5 右侧 Tab — 试玩

在编辑器里直接测试你的剧本（始终使用本地模式）：

1. 点击**"开始"**按钮启动游戏
2. 等待 AI 生成开场叙事
3. 在底部输入框写你的行动，选择行为类型后提交：
   - **思考**（紫色）：角色内心活动
   - **对话**（蓝色）：说话
   - **行动**（绿色）：做某事
   - **观察**（琥珀色）：观察环境
4. AI 会根据你的输入继续叙事
5. 可以随时点**"重置"**重新开始

> **注意**：试玩需要先在"设置"tab 配好 LLM API。没配的话会报错。

### 4.6 右侧 Tab — 调试

开发调试用，有 6 个子面板：

1. **Prompt**：查看最终组装的 system prompt 全文
2. **Messages**：查看完整对话消息列表（含 role）
3. **Tokens**：Token 分配明细
4. **State**：当前所有状态变量和值
5. **Tools**：已注册的工具和最近的工具调用记录
6. **Memory**：记忆条目和压缩事件

### 4.7 右侧 Tab — 剧本信息

编辑剧本的元数据：

#### 基本信息
- **剧本名称**：显示在首页卡片和编辑器标题栏
- **简介**：显示在首页卡片上，建议写得详细一些
- **版本号**：格式 `X.Y.Z`，每次保存自动 patch +1，也可以手动改
- **Initial Prompt**：同左侧面板底部的那个

#### 标签
- 在输入框里打字，按**回车**或点 **+** 添加标签
- 点标签右边的 **×** 删除
- 标签显示在首页卡片上，帮助玩家了解剧本类型

#### 状态变量
- 定义剧本用到的状态变量（如 chapter、stage、mood 等）
- 每行包含：变量名 | 类型 | 初始值 | 描述
- 类型支持：number、string、boolean、array、object
- 点"+ 添加变量"增加新变量，点 × 删除

#### 记忆配置
- **Token 预算**：总的上下文窗口预算
- **压缩阈值**：对话记忆超过这个 token 数时触发压缩
- **最近保留条数**：压缩时保留最近几条不压缩

#### 启用工具
- 勾选 AI 可以使用的工具（详见下方"工具系统详解"章节）：
  - `read_state`：读取状态变量
  - `query_changelog`：查询状态变更日志
  - `pin_memory`：标记重要记忆不被压缩
  - `query_memory`：搜索历史记忆
  - `set_mood`：设置角色情绪

### 4.8 右侧 Tab — 设置

配置 LLM API 端点：

#### 文本生成模型（必填）
- **Provider**：openai-compatible 或 anthropic
- **Base URL**：API 地址（如 `https://api.deepseek.com`）
- **API Key**：你的 API 密钥（有显示/隐藏切换）
- **Model**：模型名称（如 `deepseek-chat`）

#### 向量嵌入模型（可选）
- 用于记忆的语义搜索，勾选启用后出现同样的配置字段
- 如果不启用，记忆搜索功能会受限

#### 服务器同步（仅远程模式，管理员可见）
- **"从服务器拉取"**：拉取服务器上的 LLM 配置（含 API Key），覆盖本地设置
- **"同步到服务器"**：把本地配置推送到服务器，后续新会话都会用这个配置

### 4.9 发布剧本

- 顶栏右侧有**"发布"**按钮（绿色）
- 必须先保存过一次（有 script ID）才能发布
- 发布后按钮变为灰色"已发布"，再点一次可以取消发布
- **远程模式**：发布会把完整剧本数据推送到后端服务器
- **本地模式**：在本地 IndexedDB 中标记为已发布

## 五、游玩页面

点击首页卡片进入：

- 顶部显示剧本名称和返回按钮
- 如果剧本配了开场消息（openingMessages），会先显示几条静态文本
- AI 生成的叙事内容以流式方式逐字显示
- 玩家输入方式和编辑器"试玩"tab 一样（思考/对话/行动/观察）
- **远程模式**：通过 WebSocket 连接后端，后端负责调 LLM API
- **本地模式**：浏览器直接调 LLM API

## 六、工具系统详解

引擎为 AI 提供了一套工具（Tools），AI 在生成叙事时可以主动调用它们来管理游戏状态、记忆、UI 等。你在编辑器"剧本信息"tab 的"启用工具"中可以选择开放哪些工具给 AI 使用。

### 必选工具（始终启用，不可关闭）

#### `update_state` — 更新游戏状态

AI 用这个工具来修改状态变量。比如玩家到达第 2 章时，AI 会调用 `update_state({"chapter": 2})`。

- **用途**：推进剧情进度、记录玩家选择、更新角色属性等
- **参数**：一个 JSON 字符串，包含要更新的变量名和新值
- **编剧需要做的**：在"状态变量"中定义好变量（如 chapter、stage、trust_level），AI 会根据你的 prompt 来决定何时修改它们
- **提示**：你可以在 GM prompt 里写清楚"当玩家完成 XX 时，将 stage 更新为 3"之类的指示

#### `signal_input_needed` — 等待玩家输入

AI 生成一段叙事后，调用这个工具暂停并等待玩家输入。这是引擎核心循环的关键。

- **用途**：叙事到达一个需要玩家做出反应的节点时触发
- **可选功能**：AI 可以提供几个建议选项（choices），玩家可以点选或自由输入
- **编剧需要做的**：在 GM prompt 里告诉 AI 在什么时机暂停等待输入，比如"每次对话轮到玩家说话时"

### 可选工具（在"启用工具"中勾选）

#### `read_state` — 读取状态变量

AI 需要查看当前游戏状态时调用。

- **用途**：AI 在生成叙事前先确认当前章节、玩家状态等
- **参数**：可以指定读取哪些变量，或不传参数读取全部
- **推荐启用**：几乎所有剧本都应该开启，AI 需要知道当前状态才能生成合理叙事

#### `query_changelog` — 查询状态变更日志

查看某个状态变量的历史变化。

- **用途**：AI 需要回顾"之前发生了什么"时使用，比如"玩家在第几轮选择了 XX"
- **参数**：可按变量名、回合范围筛选
- **推荐启用**：较复杂的剧本（有分支记忆、需要回顾历史决策）建议开启

#### `pin_memory` — 钉住重要记忆

标记一段内容为"重要记忆"，在记忆压缩时不会被丢弃。

- **用途**：关键剧情转折点、重要的角色承诺、核心线索等，AI 会主动把它们钉住
- **参数**：要记住的内容文本 + 可选标签
- **推荐启用**：有长期叙事线、需要 AI 记住关键信息的剧本
- **提示**：你可以在 prompt 里告诉 AI "当出现重要线索时，使用 pin_memory 记住它"

#### `query_memory` — 搜索记忆

用关键词搜索历史记忆。

- **用途**：AI 在引用之前的事件前先查证，避免编造不存在的情节
- **参数**：搜索关键词
- **推荐启用**：配合 `pin_memory` 使用效果最好
- **提示**：你可以在 prompt 里告诉 AI "在提到过去的事件时，先用 query_memory 确认"

#### `set_mood` — 设置场景氛围

设置当前场景的情绪/氛围标签。

- **用途**：UI 可以根据氛围标签调整视觉效果（目前主要是元数据记录）
- **参数**：氛围标签，如 "tense"、"melancholy"、"peaceful"
- **推荐启用**：希望有氛围变化的剧本

### 编辑器中不可勾选但引擎内置的工具

以下工具在引擎内部注册，但当前版本的编辑器 UI 中没有勾选入口：

#### `inject_context` — 注入世界知识文档

AI 按需加载一篇世界观/角色设定文档到当前上下文。

- **用途**：有大量世界观文档但不能全部塞进 prompt 时，AI 可以按需加载相关的那篇
- **参数**：文档 ID

#### `list_context` — 列出可用文档

列出所有可注入的 context 类文档。

- **用途**：配合 `inject_context` 使用，AI 先查看有哪些文档可用

#### `show_image` — 显示图片/CG

在 UI 中展示一张图片。

- **用途**：在叙事关键节点展示 CG 或场景图
- **参数**：图片资源 ID
- **注意**：目前 UI 端的图片渲染功能尚未完善

### 工具在 prompt 中的引用

你可以在编剧文档（.md 文件）中用 `{{tool:工具名}}` 的格式引用工具，例如：

```markdown
当玩家做出重要选择时，使用 {{tool:update_state}} 更新相关变量。
如果玩家提到过去的事件，先用 {{tool:query_memory}} 确认细节。
```

这些标记会帮助 AI 理解在什么情境下应该使用哪些工具。

## 七、典型工作流

### 编剧创建新剧本

1. 进入编辑器
2. 上传你写好的 .md 文件（GM prompt、世界观、角色设定等）
3. 在文件列表里检查每个文件的 Role 和 Priority 是否正确，不对就手动调
4. 切到"剧本信息"tab，填写名称、简介、标签
5. 检查/调整状态变量和记忆配置
6. 切到"设置"tab，配置 LLM API
7. 切到"试玩"tab，点"开始"测试
8. 测试满意后，点"保存"，再点"发布"

### 更新已发布剧本

1. 在编辑器打开已有剧本
2. 修改文件内容或设置
3. 点"保存"（版本号自动 +1）
4. 远程模式下重新点"发布"推送到服务器

## 八、已知限制和未完成功能

以下功能在 UI 上可能可见，但**尚未完成或未完善**：

### 代码编辑器高亮（未完成）
- 当前编辑器支持基础的 Markdown 编辑
- `{{tool:xxx}}`、`{{state:xxx}}` 等特殊标记**还没有**语法高亮或彩色标签显示
- 你可以正常写这些标记，引擎能正确识别，只是编辑器里不会有视觉提示

### 自动补全（未完成）
- 在编辑器中输入 `/` 目前有一个基础的工具插入菜单
- 但完整的 `{{state:` 触发状态变量补全等功能还没做
- 你需要手动输入完整的标记

### 编辑器左右分栏预览（未完成）
- 目前没有实时大纲、工具引用列表、token 统计的侧边预览
- 这些信息可以在右侧"Prompt 预览"和"调试"tab 里查看

### 封面图上传
- 首页卡片支持显示封面图（作为卡片背景），但编辑器里**目前没有上传封面图的入口**
- 如果你通过 JSON 导入的方式在 manifest 里设置了 `coverImage` URL，是可以显示的

### 流程图编辑器
- ReactFlow 流程图组件已集成，但在当前编辑器主界面中**没有入口**
- 这是后续版本会整合的功能

## 十、常见问题

**Q: 试玩时报错"网络错误"或无响应？**
A: 检查"设置"tab 里的 LLM API 配置是否正确，特别是 Base URL 和 API Key。

**Q: 发布按钮是灰色的不能点？**
A: 需要先点"保存"，让剧本有一个 ID 之后才能发布。

**Q: 远程模式下看不到"编辑器"按钮？**
A: 需要先用 Ctrl+Shift+L 登录管理员账号。

**Q: 下架的剧本数据丢失了吗？**
A: 远程模式下，下架 = 从服务器删除。但你本地编辑器里还有数据，重新保存+发布即可恢复。

**Q: 版本号怎么控制？**
A: 每次保存自动 patch +1（如 0.0.1 → 0.0.2）。你也可以在"剧本信息"tab 手动改成任意版本号。
